Building a project from a composition#

Do the same thing as incrementally layering, but use a composition to alleviate the issues.

Issues to resolve:

  • similar prompts

    • full_name vs. author_name

  • Duplicate files:

  • Unused files

    • .releaserc.yml

Create a base composition#

A composition is a series of template layers. We can reproduce the results of the previous tutorial using this file:

my-package-composition.yaml#
1template: https://github.com/rwxd/cookiecutter-github-project
2---
3template: https://github.com/kragniz/cookiecutter-pypackage-minimal

We didn’t change the default for the project_slug, topics or open_source_license prompts. You don’t have to change all the prompts or include all the prompts in the composition.

Then run:

$ cookie-composer create /path/to/my-package-composition.yaml

email [rwxd@pm.me]: demo_user@example.com

Providing better defaults#

The composition can also improve the defaults for the template prompts by:

  • Providing new defaults that work better for your use case.

  • Using a default from a previous template.

Let’s start by changing some defaults to the first template. You can find the original prompt in this template’s cookiecutter.json file.

my-package-composition.yaml#
 1template: https://github.com/rwxd/cookiecutter-github-project
 2context:
 3  full_name: Your Name
 4  email: "{{ cookiecutter.full_name.lower().replace(' ', '_') }}@example.com"
 5  github_username: "{{ cookiecutter.full_name.lower().replace(' ', '_') }}"
 6  homepage: "https://{{ cookiecutter.github_username }}.github.io/{{ cookiecutter.project_slug }}"
 7  project_short_description: ""
 8  project_type: python
 9---
10template: https://github.com/kragniz/cookiecutter-pypackage-minimal

The new defaults provide better suggestions based on previous prompt answers.

Using values from previous templates#

The second template has several prompts that simply synonymns of promts from the previous template. It would be nice if we could use our answers from that template here.

Let’s do that. You can find the original prompts in this template’s cookiecutter.json file.

my-package-composition.yaml#
 1template: https://github.com/rwxd/cookiecutter-github-project
 2context:
 3  full_name: Your Name
 4  email: "{{ cookiecutter.full_name.lower().replace(' ', '_') }}@example.com"
 5  github_username: "{{ cookiecutter.full_name.lower().replace(' ', '_') }}"
 6  homepage: "https://{{ cookiecutter.github_username }}.github.io/{{ cookiecutter.project_slug }}"
 7  project_short_description: ""
 8  project_type: python
 9---
10template: https://github.com/kragniz/cookiecutter-pypackage-minimal
11context:
12  author_name: "{{ cookiecutter.full_name }}"
13  author_email: "{{ cookiecutter.email }}"
14  package_name: "{{ cookiecutter.project_slug }}"
15  package_description: "{{ cookiecutter.project_short_description }}"
16  package_url: "https://github.com/{{ cookiecutter.github_username }}/{{ cookiecutter.project_slug }}"
17  readme_travis_badge: false
18  readme_travis_url: ""

Now:

  • author_name defaults to full_name’s value

  • author_email defaults to email’s value

  • package_name defaults to project_slug’s value

  • package_description defaults to project_short_description’s value

  • package_url references both github_username’s and project_slug’s values
